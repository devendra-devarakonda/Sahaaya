## ğŸ¯ Key Design Decisions

### 1. Separate SELECT Policies
**Decision:** Two SELECT policies (own vs others)  
**Reason:** Clean separation of "My Requests" vs "Browse Requests"  
**Benefit:** Users never see their own requests in Browse

### 2. Real-time Subscriptions
**Decision:** Use Supabase Realtime with channel filtering  
**Reason:** Instant updates without polling  
**Benefit:** Better UX, reduced server load

### 3. Auto user_id Injection
**Decision:** Set user_id = auth.uid() in RLS policy  
**Reason:** Prevent user_id spoofing  
**Benefit:** Guaranteed data security

### 4. Service Layer Pattern
**Decision:** Create supabaseService.ts abstraction  
**Reason:** Centralize all database operations  
**Benefit:** Easier to maintain, test, and modify

### 5. Local State + Real-time
**Decision:** Combine initial query with real-time updates  
**Reason:** Fast initial load + live updates  
**Benefit:** Best of both worlds
